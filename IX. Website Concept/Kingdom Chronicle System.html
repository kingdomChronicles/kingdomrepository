Kingdom Chronicle System



USER ACCOUNTS


USER ACCOUNT (approved)

  1. Approved accounts after the orientation and has already started in his devotional activity.
  2. Can received personal messages from mentor and admin accounts.
  3. No user to user messages.

USER ACCOUNT (pending)

  1. A referred User account by a member.
  2. Pending User Account is an referred account by a member.
  3. No user to user messages.
  4. Can only View the global Reflection Page.
  5. Can sort reflection page by date/age.
  6. Can Visit Written Testimonial Portion Page.


MENTOR ACCOUNTS


MENTOR ACCOUNT (apprentice)

  1. Upgraded user account.
  2. Can received personal messages from mentor and admin accounts.
  3. Oversee all the interactions of mentor and users account, and oversee all the progress of the users accounts.
  4. No interactions with the users accounts. Only view the activities of Mentor Account and User Account


MENTOR ACCOUNT (Field)

  1. Upgraded Mentors Account (Apprentice).
  2. Can received personal messages from admin accounts.
  3. They are the overseer of a group of user accounts. A certain count of user accounts
  4. Monitor all the activity of user accounts under his network.
  5. Can send personal message to a certain user account on his network
  6. Can send group message for all his user account under his network
  7. Can send personal message to a certain Mentor account (Apprentice) on his network
  8. Can send group message for all his Mentor account (Apprentice) under his network
  9. The one to recommend a user for an upgrade as Mentors Account (Apprentice)
  10. The one to recommend a Mentors Account (Apprentice) for an upgrade as Mentors Account (Field).


ADMINISTRATION ACCOUNTS


MENTOR ACCOUNT (Admin)

  1. Upgraded Mentor Account (Field).
  2. Oversee all the interactions of Mentors Account (Field ) and Mentors Account (Apprentice)
  3. Oversee all the interactions of mentors and users account
  4. Can send personal message to a certain user account on his network
  5. Can send group message for all user account under his network
  6. Can send personal message to a certain Mentor account (Apprentice) on his network
  7. Can send group message for all Mentor account (Apprentice) under his network
  8. Can send personal message to a certain Mentor account (Field) on his network
  9. Can send group message for all Mentor account (Field) under his network
  10. The one to approve all recommendations of all upgrades of User Accounts to Mentor Accounts (Apprentice)
  11. The one to approve all recommendations of all upgrades of Mentor Accounts (Apprentice) to Mentor Accounts (Field)


MENTOR ACCOUNT (SuperAdmin)

  1. Upgraded Mentor Account (Admin).
  2. Oversee all the interactions of Mentors Account (Field ) and Mentors Account (Apprentice)
  3. Oversee all the interactions of mentors and users account
  4. Can send personal message to a certain user account on his network
  5. Can send group message for all user account under his network
  6. Can send personal message to a certain Mentor account (Apprentice) on his network
  7. Can send group message for all Mentor account (Apprentice) under his network
  8. Can send personal message to a certain Mentor account (Field) on his network
  9. Can send group message for all Mentor account (Field) under his network
  10. Can send personal message to a certain Mentor account (Admin) on his network
  11. Can send group message for all Mentor account (Field) under his network
  12. The one to approve all recommendations of all upgrades of Mentor Accounts (Field) to Mentor Accounts (Admin)


FINACIAL MONITORING ACCOUNT (SuperAdmin)

  1. incharge in monitoring and managing, financial donations and pledges from users account, affiliate institutions and religious institutions.


MANAGEMENT ACCOUNT (SuperAdmin)

  1. All Features of Mentor Account (SuperAdmin)
  2. Approval and Oversee all activity of Financial Monitoring Account
  3. Approval and Oversee all the Features of all Religious/Churches Accounts


PHASE 2 ACCOUNTS


Affiliate Accounts (Non Religious Affiliates)

  1. this are organizations and institutions who adopts the services of the program.
  2. They can monitor the progress of each user accounts that is part of their institutions only.
  3. View is limited to their own institution only


Religious Accounts

  1. this are Religious groups who adopts the services of the program.
  2. They can monitor the progress of each user accounts that is part of their institutions only.
  3. View is limited to their own institution only
  4. Additional Features: Automated attendance in the Chuch
  5. Additional Features: Financial Reports
  6. Electronic Offering w/c includes even their members abroad


Voice Call Features

  1. All Mentors Account Except Mentor's Account (Apprentice)


USER ACTIVITY FLOW


Registration to Orientation Schedule

  1. A valid user account (all account) will send a sharable link were a prospect referal may fill-up his registration form
  2. Referal Link was send thru other platforms like, messenger, messages etc.
  3. New user will fill up registration form for New User Accounts
  4. New user account will appear on the page of the Mentor Account (Admin)
  5. Mentor Account (Admin) will call/screened the new user. And Send her a personal link were her page activity is present.
  6. The moment the admin screened the new user account, a schedule oriention will appear on his/her page. they can choose different orientation schedules they prefer according to their preferences. Or they may create preferred time start of orientation but on Admin's approval so that it will be add up to the list of orientation schedules of the week.
  7. Orientation Schedule is set from monday to saturday anytime.
  8. Mentor Account Admin will send a link for the personal page of the new user
  9. New user account status is set to Pending...,
  10. The New User can now view the Global Chat Group for Reflections
  11. New User can also view the Written Testimonial Page


Orientation to Devotional Activity

  1. The New User will attend a scheduled Batch Virtual orientation base on Her choice Batch available schedule
  2. Mentor Account (Admin) will assign the new user to a Mentor Account (Field)
  3. Mentor Account (Admin) will now approved the New User Account
  4. A welcome message will appear on the New User page
  5. 12 Midnight Sunday, Module 1 Day 1 will appear on Her page


Devotional Activity Module 1 Day 1 to to Module 2

  1. The User will click the Day 1 Lesson Button
  2. Reflection Duration Time Start
  3. User Account will read the instruction, read the Verses, read the Guide Reflection,
    Sample Devotion Lesson: Module 1 / Day 1 w/o video
    Sample Devotion Lesson: Module 6 / Day 4 am w/ Picture
    Sample Devotion Lesson: Module 3 / Day 6 pm w/ video
  4. User will Click Write my Reflection Now
  5. After writing Her own reflection from her devotion, click submit button, Duration Time End
  6. When submitted, she will be redirected to the reflection chat page, her default is the list of all reflections of that same topic sorted on date, she was the latest to submit, reflected pang-ilan na siya sa lahat ng nag submit.
  7. Sorting available: base on date/rating/age
  8. Previous reflections can be viewed and sorted base on ratings
  9. Module 1 Day 2 will be available at 12 midnight Monday, repeat activity flow till reflection is submitted
  10. Module 1 Day 3 am will be available at 12 midnight Tuesday, repeat activity flow till reflection is submitted
  11. Module 1 Day 3 pm will be available at 12 nn Wednesday, repeat activity flow till reflection is submitted
  12. Module 1 Day 4 am will be available at 12 midnight Wednesday, repeat activity flow till reflection is submitted
  13. Module 1 Day 4 pm will be available at 12 nn Thursday, repeat activity flow till reflection is submitted
  14. Module is a set of 1 week Lessons, with am and pm lessons


Devotional Activity Module 1 Day 1 to to Module 2

  1. The User will click the Day 1 Lesson Button
  2. Reflection Duration Time Start
  3. User Account will read the instruction, read the Verses, read the Guide Reflection
  4. User will Click Write my Reflection Now
  5. After writing Her own reflection from her devotion, click submit button, Duration Time End
  6. Finished Lessons button will set to read Only and not clickable
  7. When submitted, she will be redirected to the reflection chat page, her default is the list of all reflections of that same topic sorted on date, she was the latest to submit, reflected pang-ilan na siya sa lahat ng nag submit.
  8. Sorting available: base on date/rating/age
  9. Previous reflections can be viewed and sorted base on ratings
  10. Module 1 Day 2 will be available at 12 midnight Monday, repeat activity flow till reflection is submitted
  11. Module 1 Day 3 am will be available at 12 midnight Tuesday, repeat activity flow till reflection is submitted
  12. Module 1 Day 3 pm will be available at 12 nn Wednesday, repeat activity flow till reflection is submitted


Activity Attendance and Late submissions

  1. Activity Attendance is set to present or done the moment a user submitted her reflection
  2. Module 1 Day 1 submission is set until 12 midnight Monday
  3. Module 1 Day 2 submission is set until 12 midnight Tuesday
  4. Starting Day 3, Module 1 Day 3 am submission is until 12 noon Wednesday
  5. Module 1 Day 3 pm submission is until 12 Midnight Thursday
  6. When reflection is submitted late from the set time, it will be mark as late. Count of Late is 1 demerit
  7. When reflection is unsubmitted from the set time, it will be mark as absent. count of absent is 2 demerit
  8. A total accumulated 4 demerits will set the users lesson back to Module 1 Day 1 the following Monday
  9. Absent and late must appear a notification to the user up to the admin account
  10. Mentor may recommend a special account, if user is on a valid reason like no data or slow internet reasons.
  11. A graph of Users daily activity submission is needed, as part of mentors basis for recomendation for an upgraded account.
